# 🛡️ RASP Module for Linux x86_64

**Runtime Application Self-Protection** — динамическая библиотека для защиты Linux-приложений от ROP-атак (Return-Oriented Programming) без модификации исходного кода.

## 📌 О проекте

Данный проект разработан в рамках школьной проектной работы по профилю «Информационная безопасность» (11 класс).

**Проблема:** Классические методы защиты (Stack Canary, ASLR, NX) не всегда эффективны против ROP-атак, особенно на Legacy-системах, где невозможно перекомпилировать приложение.

**Решение:** Модуль внедряется в процесс через `LD_PRELOAD` и перехватывает опасные вызовы (`system`, `execve`), анализируя целостность потока управления (Control Flow Integrity) по алгоритму «Smart Stack Walk».

**Результат:** Блокировка 100% тестовых ROP-эксплойтов с задержкой менее 2 мс.

---

## ✨ Ключевые возможности

- ✅ **Без изменения кода** — защита работает на скомпилированных бинарниках
- ✅ **Динамическое картирование памяти** — корректно работает с ASLR и PIE
- ✅ **Фильтрация системных вызовов** — отсутствие ложных срабатываний
- ✅ **Логирование атак** — детальный вывод в консоль для анализа
- ✅ **User-mode** — безопасен для системы (ошибка не вызовет Kernel Panic)

## 🚀 Быстрый старт

##1. Клонирование репозитория
bash
git clone https://github.com/your-username/rasp-module.git
cd rasp-module

##2. Компиляция
bash

gcc -shared -fPIC -o libdefense.so libdefense_pro.c -ldl

##3. Запуск с защитой
bash

LD_PRELOAD=./libdefense.so ./your_application

Пример лога успешной блокировки:

[RASP DEBUG] Анализ стека (поиск User Code)...
 [0] 0x7f8a4b2a1c3d -> USER CODE FOUND!
 [FAIL] Инструкции CALL нет! (Байт: 0xc3)
RASP ALERT! ROP-АТАКА ЗАБЛОКИРОВАНА!

⚠️ Ограничения
Ограничение	Описание
Только динамическая линковка	Не работает со статическими бинарниками (-static)
Только x86_64	Не поддерживает ARM или x86
Зависимость от ABI	Может давать сбои на Go/Rust приложениях
Некритичная нагрузка	+5-7% CPU при интенсивных вызовах system
🛠️ Технологии

    Язык: C99

    API: dl_iterate_phdr, backtrace, dlsym

    Формат: ELF, System V AMD64 ABI

    Сборка: GCC + Make
